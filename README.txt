# قناص (Qanas) - أداة اختبار اختراق متقدمة

## نبذة عن الأداة

Qanas (قناص) هي أداة اختبار اختراق متقدمة ومنصة لإدارة سطح الهجوم، مصممة لأتمتة عمليات جمع المعلومات واكتشاف الثغرات الأمنية. تجمع الأداة بين العديد من الأدوات المفتوحة المصدر والتجارية لتوفير حل شامل لاختبار الاختراق.

## المميزات

- **جمع المعلومات**: جمع معلومات مفصلة عن الهدف بشكل آلي
  - فحص المنافذ باستخدام Nmap
  - استعلامات WHOIS للنطاقات
  - استعلامات DNS وتعداد النطاقات الفرعية
  - اكتشاف التقنيات المستخدمة

- **فحص الثغرات**: اكتشاف وتحليل الثغرات الأمنية في الأنظمة والتطبيقات
  - فحص تطبيقات الويب
  - فحص الخدمات والبروتوكولات
  - اكتشاف الإعدادات الخاطئة

- **إدارة سطح الهجوم**: مراقبة وتحليل سطح الهجوم بشكل مستمر
  - تتبع الأصول والخدمات
  - تحديد نقاط الضعف المحتملة

- **تقارير مفصلة**: إنشاء تقارير مفصلة عن نتائج الاختبارات
  - تقارير HTML تفاعلية
  - تصدير النتائج بتنسيقات متعددة

- **واجهة سهلة الاستخدام**: واجهة سطر أوامر بسيطة وفعالة

## أوضاع التشغيل

- **الوضع العادي (Normal Mode)**: فحص هدف واحد بشكل مفصل. يقوم بإجراء فحص شامل للهدف المحدد (موقع ويب أو عنوان IP) ويجمع معلومات مفصلة عنه.

- **وضع AIRSTRIKE**: فحص سريع لعدة أهداف من ملف. يقوم بإجراء فحص سريع لكل هدف في الملف المحدد.

- **وضع NUKE**: فحص شامل لعدة أهداف من ملف. يقوم بإجراء فحص شامل ومكثف لكل هدف في الملف المحدد.

- **وضع LOOT**: تنظيم وعرض نتائج الفحص السابقة.

## المتطلبات

### متطلبات النظام
- نظام تشغيل Windows أو Linux (Ubuntu/Debian/Kali)
- Python 3.8+

### الأدوات الخارجية المطلوبة
لاستخدام جميع ميزات الأداة، يجب تثبيت الأدوات التالية:
- nmap: أداة فحص الشبكات
- whois: أداة استعلام عن معلومات النطاقات
- host: أداة استعلام DNS

#### تثبيت الأدوات على نظام Windows
يمكن تثبيت الأدوات المطلوبة على نظام Windows باستخدام أحد مديري الحزم التالية:

1. **باستخدام Chocolatey**:
```
choco install nmap whois bind-toolsonly
```

2. **باستخدام Scoop**:
```
scoop install nmap whois bind
```

## التثبيت

### على نظام Windows

1. قم بتنزيل المشروع أو استنساخه من GitHub:
```
git clone https://github.com/SaudiLinux/Qanas.git
```

2. انتقل إلى مجلد المشروع:
```
cd Qanas
```

3. قم بتثبيت متطلبات Python:
```
pip install -r requirements.txt
```

4. تأكد من إنشاء المجلدات المطلوبة:
```
mkdir loot
```

### على نظام Linux

1. قم بتنزيل المشروع أو استنساخه من GitHub:
```
git clone https://github.com/SaudiLinux/Qanas.git
```

2. انتقل إلى مجلد المشروع:
```
cd Qanas
```

3. قم بتشغيل سكربت التثبيت:
```
chmod +x install.sh
./install.sh
```

## الاستخدام

### على نظام Windows

```
python qanas.py example.com                # فحص هدف واحد (موقع ويب أو عنوان IP)
python qanas.py C:\path\to\targets.txt airstrike  # فحص عدة أهداف من ملف (وضع AIRSTRIKE)
python qanas.py C:\path\to\targets.txt nuke      # فحص شامل لعدة أهداف (وضع NUKE)
python qanas.py loot                      # عرض نتائج الفحص (وضع LOOT)
python qanas.py --version                 # عرض إصدار الأداة
```

### على نظام Linux

```
./qanas example.com                # فحص هدف واحد (موقع ويب أو عنوان IP)
./qanas /path/to/targets.txt airstrike  # فحص عدة أهداف من ملف (وضع AIRSTRIKE)
./qanas /path/to/targets.txt nuke      # فحص شامل لعدة أهداف (وضع NUKE)
./qanas loot                      # عرض نتائج الفحص (وضع LOOT)
./qanas --version                 # عرض إصدار الأداة
```

## ملاحظات هامة

1. **الأدوات الخارجية**: إذا لم تكن الأدوات الخارجية (nmap, whois, host) مثبتة، ستعمل الأداة بوظائف محدودة وستقوم بتسجيل رسائل الخطأ في مجلد النتائج.

2. **مجلد النتائج**: يتم حفظ جميع نتائج الفحص في مجلد `loot` داخل مجلد المشروع.

3. **تشغيل الأداة كمسؤول**: بعض وظائف الفحص قد تتطلب صلاحيات المسؤول، خاصة على نظام Windows.

## استكشاف الأخطاء وإصلاحها

1. **خطأ "Command not found"**: تأكد من تثبيت الأدوات الخارجية (nmap, whois, host) وإضافتها إلى متغير PATH.

2. **مشاكل في الوصول إلى المجلدات**: تأكد من وجود جميع المجلدات المطلوبة (loot, modules, templates, plugins, assets, wordlists).

3. **أخطاء Python**: تأكد من تثبيت جميع متطلبات Python المذكورة في ملف requirements.txt.

4. **فشل عمليات المسح**: إذا فشلت عمليات المسح، تحقق من ملفات السجل في مجلد النتائج للحصول على معلومات حول الأخطاء.

5. **مشاكل الأداء**: إذا كانت عمليات المسح بطيئة، حاول تقليل نطاق المسح أو استخدام وضع AIRSTRIKE بدلاً من NUKE.

للمزيد من المعلومات حول استكشاف الأخطاء وإصلاحها، يرجى الرجوع إلى ملف TROUBLESHOOTING.txt.

## التوثيق

للحصول على معلومات أكثر تفصيلاً، يرجى الرجوع إلى الملفات التالية:

- **QUICK_START.txt**: دليل البدء السريع
- **USAGE_GUIDE_AR.txt**: دليل استخدام مفصل
- **CONFIG_GUIDE_AR.txt**: دليل تكوين الأداة
- **TROUBLESHOOTING.txt**: استكشاف الأخطاء وإصلاحها
- **SECURITY.txt**: سياسة الأمان وإرشادات الإبلاغ عن الثغرات
- **CHANGELOG.txt**: سجل التغييرات والتحديثات
